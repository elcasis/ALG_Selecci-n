01 · Fundamentos

¿Qué es y por qué importa?

Cuatro ideas clave que definen el algoritmo de selección y lo diferencian de los demás.

🔍

La idea central: buscar el mínimo

En cada pasada, escanea todo el tramo no ordenado [i..n-1] para encontrar el índice del elemento más pequeño. Luego lo mueve a la posición i con un único intercambio.

SELECCIÓN DE MÍNIMO
🔄

Solo n-1 intercambios

Su ventaja clave: hace como máximo n-1 intercambios — uno por pasada, solo cuando el mínimo no está ya en su sitio. Ideal cuando escribir en memoria es costoso (flash, EEPROM).

ESCRITURAS MÍNIMAS
📊

Sin optimización posible

A diferencia de burbuja, no puede parar antes. Aunque el array ya esté ordenado, sigue haciendo las n(n-1)/2 comparaciones. Su complejidad es fija en todos los casos.

COMPLEJIDAD CONSTANTE
⚠️

No es estable

El intercambio puede cambiar el orden relativo de elementos iguales. Ejemplo: en [3a, 3b, 1], el 1 se intercambia con 3a, quedando [1, 3b, 3a] — el orden de los 3 se invirtió.

INESTABILIDAD
02 · Mecánica

Los 4 pasos del algoritmo

El bucle exterior fija posiciones, el interior busca. Así se combinan para ordenar.

01

Fijar posición i

El bucle exterior controla la posición i que se va a rellenar (de 0 a n-2). Todo lo anterior a i ya está ordenado definitivamente.

02

Asumir mínimo en i

Al inicio de cada pasada, idx_min = i. Asumimos provisionalmente que el mínimo ya está en la posición actual. El escaneo puede actualizarlo.

03

Escanear [i+1..n-1]

El bucle interior compara cada elemento con arr[idx_min]. Si encuentra algo menor, actualiza idx_min = j. Al final, idx_min apunta al mínimo real.

04

Intercambiar si hace falta

Solo intercambia si idx_min ≠ i. Si el mínimo ya estaba en i, no hace nada. Esto garantiza que los intercambios sean siempre los mínimos posibles.

05 · Código

Pseudocódigo completo

Implementación canónica con anotaciones en cada bloque lógico.

   seleccion.pseudo  —  versión con intercambio condicional 
función seleccion(arr):
  n = longitud(arr)

  // ── Bucle exterior: n-1 pasadas, una por posición ────────────────────
  para i desde 0 hasta n-2:

    // ── Asumir que el mínimo está en la posición i ───────────────────
    idx_min = i

    // ── Escanear el tramo no ordenado para encontrar el mínimo real ──
    para j desde i+1 hasta n-1:
      si arr[j] < arr[idx_min]:
        idx_min = j         // actualizar índice del mínimo

    // ── Intercambiar solo si el mínimo no estaba ya en i ─────────────
    si idx_min ≠ i:
      temp          = arr[i]
      arr[i]        = arr[idx_min]
      arr[idx_min]  = temp

  devolver arr


// ── Reconstruir el camino del mínimo por pasada ──────────────────────
// Pasada 1: min=[64,25,12,22,11,90,38] → 11 (idx 4) → swap con pos 0
// Pasada 2: min=[25,12,22,64,90,38]    → 12 (idx 1) → swap con pos 1 (no hace falta)
// Pasada 3: min=[22,64,90,38]          → 22 (idx 0) → ya en su sitio
// ...
06 · Análisis

Complejidad temporal y espacial

Por qué tiene complejidad constante y cuándo eso es una ventaja.

Mejor caso
O(n²)
Aunque el array esté ordenado, sigue comparando todos los pares. No hay forma de detectarlo sin comparar.
Caso medio
O(n²)
Siempre realiza exactamente n(n-1)/2 comparaciones. Con n=7: 21 comparaciones fijas.
Peor caso
O(n²)
Idéntico al mejor. La complejidad de comparaciones es siempre cuadrática, sin excepción.
Intercambios
O(n)
Máximo n-1 intercambios — uno por pasada, solo cuando es necesario. Su gran ventaja sobre burbuja.
Cuándo usarlo
✅

Cuándo SÍ usar selección

— Memoria flash / EEPROM (escrituras costosas)
— Arrays pequeños donde el código simple importa
— Cuando necesitas exactamente n-1 intercambios
— Aprender la diferencia entre comparaciones e intercambios
— Sistemas embebidos con restricciones de escritura

❌

Cuándo NO usar selección

— Arrays grandes (n > 1000)
— Cuando necesites estabilidad (usa MergeSort)
— Arrays casi ordenados (usa Inserción o Burbuja)
— Cuando la velocidad total importa más que los intercambios
— Producción con datos de tamaño variable
