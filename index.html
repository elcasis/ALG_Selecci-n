Algoritmos de Ordenación

Selección
Ordenado.

Aprende cómo el algoritmo de selección busca el mínimo en cada pasada y lo coloca en su posición correcta.

🔍

La idea central

En cada pasada, busca el elemento más pequeño del tramo sin ordenar y lo coloca al inicio de ese tramo.

🔄

Intercambios mínimos

Solo hace n-1 intercambios como máximo, uno por pasada. Mucho menos que burbuja en datos desordenados.

📊

Complejidad fija

Siempre hace O(n²) comparaciones, sin importar si el array está casi ordenado. No hay optimización posible.

🧠

¿Dónde se usa?

Útil cuando las escrituras en memoria son costosas (flash, EEPROM). Minimiza intercambios al máximo.

¿Cómo funciona?

01

Posición actual

Fija una posición i (empieza en 0). El objetivo es encontrar qué elemento debería ir aquí.

02

Buscar el mínimo

Escanea todo el tramo [i..n-1] para encontrar el índice del elemento más pequeño.

03

Intercambiar

Si el mínimo no estaba ya en la posición i, intercambia arr[i] con arr[min]. Un solo intercambio.

04

Avanzar

La posición i queda fijada para siempre. Avanza i y repite hasta cubrir todo el array.

Complejidad temporal

Mejor caso
O(n²)
Incluso con el array ya ordenado, sigue buscando el mínimo en cada pasada. Sin optimización.
Caso medio
O(n²)
Siempre hace n·(n-1)/2 comparaciones sin importar el estado inicial del array.
Peor caso
O(n²)
Igual que el mejor. La complejidad es constante en todos los casos.
Intercambios
O(n)
Solo n-1 intercambios máximo. Su gran ventaja frente a burbuja.

   seleccion.pseudo 
función seleccion(arr):
  n = longitud(arr)

  para i desde 0 hasta n-2:          // posición a rellenar
    idx_min = i                        // asume que el mínimo está en i

    para j desde i+1 hasta n-1:      // escanea el resto
      si arr[j] < arr[idx_min]:
        idx_min = j                   // nuevo mínimo encontrado

    si idx_min ≠ i:                  // solo intercambia si hace falta
      temp          = arr[i]
      arr[i]       = arr[idx_min]
      arr[idx_min] = temp

  devolver arr
